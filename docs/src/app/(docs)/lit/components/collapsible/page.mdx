## Anatomy

Import the custom elements and assemble the parts:

```html title="Anatomy"
<collapsible-root>
  <collapsible-trigger>Toggle section</collapsible-trigger>
  <collapsible-panel>Section content</collapsible-panel>
</collapsible-root>
```

## Usage guidelines

- Use `collapsible-trigger` as the interactive control for opening and closing the panel.
- Use `keep-mounted` when the panel content needs to stay in the DOM while hidden.
- Use `hidden-until-found` when browser find-in-page should reveal the panel automatically.

## API reference

### CollapsibleRootElement

Custom element: `<collapsible-root>`

| Attribute / Property | Type                      | Default | Description                                           |
| -------------------- | ------------------------- | ------- | ----------------------------------------------------- |
| `.open`              | `boolean \| undefined`    | —       | Controlled open state.                                |
| `.defaultOpen`       | `boolean`                 | `false` | Initial open state when uncontrolled.                 |
| `disabled`           | `boolean`                 | `false` | Whether the component should ignore user interaction. |
| `.onOpenChange`      | `(open, details) => void` | —       | Called when the open state changes.                   |

### CollapsibleTriggerElement

Custom element: `<collapsible-trigger>`

Interactive control that toggles the panel and manages `aria-expanded` / `aria-controls`.

### CollapsiblePanelElement

Custom element: `<collapsible-panel>`

| Attribute / Property | Type      | Default | Description                                                        |
| -------------------- | --------- | ------- | ------------------------------------------------------------------ |
| `keep-mounted`       | `boolean` | `false` | Keeps the panel mounted while hidden.                              |
| `hidden-until-found` | `boolean` | `false` | Uses `hidden="until-found"` so browser find-in-page can reveal it. |

### Data attributes

`<collapsible-root>`, `<collapsible-trigger>`, and `<collapsible-panel>` can receive:

- `data-open`
- `data-closed`
- `data-disabled`
- `data-starting-style`
- `data-ending-style`

`<collapsible-trigger>` also receives:

- `data-panel-open`
