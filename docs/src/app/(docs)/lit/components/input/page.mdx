## Usage guidelines

- **Form controls must have an accessible name**: Use a native `<label>` or pair the input with the Lit `field` utilities where applicable.

## Anatomy

```html title="Anatomy"
<input-root>
  <input placeholder="Email address" />
</input-root>
```

## API reference

### InputRootElement

Custom element: `<input-root>`

| Property        | Type                           | Default | Description                                            |
| --------------- | ------------------------------ | ------- | ------------------------------------------------------ |
| `disabled`      | `boolean`                      | `false` | Whether the component should ignore user interaction.  |
| `defaultValue`  | `string \| number \| string[]` | —       | The default value of the input. Use when uncontrolled. |
| `value`         | `string \| number \| string[]` | —       | The value of the input. Use when controlled.           |
| `onValueChange` | `(value, details) => void`     | —       | Callback fired when the value changes.                 |

### Data attributes

The wrapped native `<input>` or `<textarea>` receives the following attributes:

- `data-disabled`
- `data-valid`
- `data-invalid`
- `data-dirty`
- `data-touched`
- `data-filled`
- `data-focused`
