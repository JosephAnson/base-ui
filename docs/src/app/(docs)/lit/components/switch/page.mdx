## Usage guidelines

- **Form controls must have an accessible name**: It can be created using a `<label>` element or the `Field` component. See [Labeling a switch](#labeling-a-switch) and the [forms guide](/react/handbook/forms).

## Anatomy

Import the custom elements and assemble their parts:

```ts title="Anatomy"
import '@base-ui/lit/switch';

html`<switch-root>
  <switch-thumb></switch-thumb>
</switch-root>`;
```

## Examples

### Labeling a switch

An enclosing `<label>` is the simplest labeling pattern:

```ts title="Wrapping a label around a switch"
html`<label>
  <switch-root>
    <switch-thumb></switch-thumb>
  </switch-root>
  Notifications
</label>`;
```

### Rendering as a native button

Custom elements are the primary Lit API. If you need React-style polymorphism or native-button rendering, the optional helper API also supports a native-button path:

<DemoSwitchNativeButton compact />

```ts title="Sibling label pattern with a native button"
import { Switch } from '@base-ui/lit/switch';

html`
  <label for="notifications-switch">Notifications</label>
  ${Switch.Root({
    id: 'notifications-switch',
    nativeButton: true,
    render: html`<button></button>`,
    children: Switch.Thumb({}),
  })}
`;
```

### Form integration

Use [Field](/lit/components/field) to handle label associations and form integration:

```ts title="Using Switch in a form"
html`
  <form-root>
    <form>
      <field-root name="notifications">
        <field-label>
          <switch-root>
            <switch-thumb></switch-thumb>
          </switch-root>
          Notifications
        </field-label>
      </field-root>
    </form>
  </form-root>
`;
```
