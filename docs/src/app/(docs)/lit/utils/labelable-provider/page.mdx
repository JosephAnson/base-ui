# Labelable Provider

<Subtitle>Manages label-control ID associations for accessibility.</Subtitle>
<Meta
  name="description"
  content="A provider element for managing label-control associations and accessible descriptions in Base UI Lit."
/>

## Anatomy

```html title="Anatomy"
<labelable-provider control-id="my-input">
  <label id="my-label">Name</label>
  <input id="my-input" aria-labelledby="my-label" />
  <span id="my-error">Required field</span>
</labelable-provider>
```

The provider renders with `display: contents`, keeps track of the active control and label ids, and combines description ids for `aria-describedby`.

## API reference

### LabelableProviderElement

Custom element: `<labelable-provider>`

| Attribute    | Type             | Default     | Description                                    |
| ------------ | ---------------- | ----------- | ---------------------------------------------- |
| `control-id` | `string \| null` | generated   | Explicit control id to expose through context. |
| `label-id`   | `string`         | `undefined` | Explicit label id to expose through context.   |

### getLabelableContext

Use this helper to read the nearest labelable context from a descendant element.

```ts title="Using getLabelableContext"
import { getLabelableContext } from '@base-ui/lit/labelable-provider';

const ctx = getLabelableContext(element);
if (ctx) {
  ctx.registerControlId(Symbol('my-control'), 'my-input');
  ctx.registerLabelId('my-label');
  ctx.registerMessageId('my-error', true);

  const props = ctx.getDescriptionProps({ 'aria-describedby': 'custom-help' });
  // { 'aria-describedby': 'my-error custom-help' }
}
```
