## Anatomy

Import the element and wrap it around your app:

```html title="Anatomy"
<script type="module">
  import '@base-ui/lit/direction-provider';
</script>

<direction-provider direction="rtl">
  <!-- Your app or a group of components -->
</direction-provider>
```

`<direction-provider>` enables child Base UI components to adjust behavior based on RTL text direction, but does not affect HTML and CSS. The `dir="rtl"` HTML attribute or `direction: rtl` CSS style must be set additionally by your own application code.

## API reference

### DirectionProvider

Custom element: `<direction-provider>`

| Attribute   | Type             | Default | Description                        |
| ----------- | ---------------- | ------- | ---------------------------------- |
| `direction` | `'ltr' \| 'rtl'` | `'ltr'` | The reading direction of the text. |

The element mirrors `direction` to a `dir` attribute so descendant Lit components can resolve the effective text direction from the DOM tree.

### getDirection

Use this helper to read the current text direction from any element. This is useful for components that need the resolved direction from the nearest provider or DOM ancestor, including content rendered outside the provider subtree.

```ts title="Reading the current direction"
import { getDirection } from '@base-ui/lit/direction-provider';

const direction = getDirection(element);
```
