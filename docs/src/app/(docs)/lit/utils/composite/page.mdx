## Usage

Use the composite utilities to calculate the next highlighted item for roving-focus widgets.

```ts title="List navigation"
import { navigateList } from '@base-ui/lit/composite';

const nextIndex = navigateList({
  currentIndex: highlightedIndex,
  items: listElements,
  event: keyboardEvent,
  orientation: 'vertical',
  loop: true,
  enableHomeAndEndKeys: true,
});

if (nextIndex !== -1) {
  highlightedIndex = nextIndex;
}
```

```ts title="Grid navigation"
import { navigateGrid } from '@base-ui/lit/composite';

const nextIndex = navigateGrid({
  currentIndex: highlightedIndex,
  items: gridElements,
  event: keyboardEvent,
  cols: 3,
  direction: 'rtl',
  loop: true,
});
```

## API reference

### `navigateList(options): number`

Computes the next highlighted index for a one-dimensional list.

| Option                 | Type                                   | Default      | Description                                                      |
| ---------------------- | -------------------------------------- | ------------ | ---------------------------------------------------------------- |
| `currentIndex`         | `number`                               | â€”            | Current highlighted index. Use `-1` when no item is highlighted. |
| `items`                | `ReadonlyArray<HTMLElement \| null>`   | â€”            | Ordered list of navigable items.                                 |
| `event`                | `KeyboardEvent`                        | â€”            | Keyboard event used to calculate navigation.                     |
| `orientation`          | `'horizontal' \| 'vertical' \| 'both'` | `'vertical'` | Which arrow keys should move focus.                              |
| `direction`            | `'ltr' \| 'rtl'`                       | `'ltr'`      | Enables RTL-aware horizontal navigation.                         |
| `loop`                 | `boolean`                              | `false`      | Wraps focus at list boundaries.                                  |
| `disabledIndices`      | `number[] \| ((index) => boolean)`     | â€”            | Explicitly controls which indices should be skipped.             |
| `enableHomeAndEndKeys` | `boolean`                              | `true`       | Enables `Home` and `End` navigation.                             |

Returns `-1` when the key is not a supported navigation key for the supplied options.

### `navigateGrid(options): number`

Computes the next highlighted index for a grid.

| Option                     | Type     | Default | Description                                      |
| -------------------------- | -------- | ------- | ------------------------------------------------ |
| All `navigateList` options | â€”        | â€”       | Grid navigation accepts the same shared options. |
| `cols`                     | `number` | â€”       | Number of columns in the grid.                   |

Grid navigation keeps horizontal movement within the current row and supports vertical wrapping when `loop` is enabled.

### `findNonDisabledIndex(items, options): number`

Walks forward or backward through the item list until a non-disabled index is found.

### `getMinListIndex(items, disabledIndices?): number`

Returns the first non-disabled index.

### `getMaxListIndex(items, disabledIndices?): number`

Returns the last non-disabled index.

### `isListIndexDisabled(items, index, disabledIndices?): boolean`

Checks whether an item should be treated as disabled. When `disabledIndices` is omitted, the helper falls back to DOM-disabled state via `disabled` and `aria-disabled="true"`.

### `isIndexOutOfBounds(items, index): boolean`

Checks whether an index falls outside the item list range.
